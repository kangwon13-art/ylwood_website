# Architecture & UI Decisions

## UI Breakpoint & Optimization Policy
* **Date:** 2026-07-11
* **Decision:** 모든 UI 개선 작업(여백 조정, 헤더 축소, 모바일 전용 컴포넌트 등)은 별도 명시가 없는 한 **모바일(768px 이하)과 태블릿(769px ~ 1024px)** 두 브레이크포인트 모두에 동일하게 적용한다.
* **배경:** 모바일뿐만 아니라 태블릿(아이패드 등) 환경에서도 동일하게 일관된 고밀도 콤팩트 레이아웃을 제공하기 위함이다. PC 스타일은 **1025px 이상**의 환경에서만 독립적으로 작동한다.

---

## 인라인 style 우선순위 확인 원칙
* **Date:** 2026-07-11
* **Decision:** CSS 미디어 쿼리에서 `!important`를 사용해도 HTML 마크업의 인라인 `style=""` 속성보다 우선순위가 낮다. 스타일 적용이 예상대로 되지 않을 때는 **반드시 HTML 마크업의 인라인 스타일을 먼저 점검**한다.
* **배경:** 인라인 스타일은 CSS 명세상 선택자 우선순위 계산 밖에서 가장 높은 명시도를 가진다. 실제 작업에서 `.calculator-section`과 `.global-search-container`의 인라인 스타일이 `!important` 미디어 쿼리 전부를 무력화하여 좌우 여백이 남는 현상이 반복 발생했다.
* **규칙:** 스타일 변경 시 해당 HTML 요소에 인라인 `style=`이 존재하는지 grep으로 먼저 확인한 후 작업한다.

## 로컬 개발 서버 캐시 정책 (no-store)
* **Date:** 2026-07-11
* **Decision:** 로컬 개발용 Python HTTP 서버는 `Cache-Control: no-store` 헤더를 반드시 포함하여 서빙한다. 실기기(모바일)에서 확인 시 파일 수정 후 반드시 브라우저 강제 새로고침 또는 캐시 버스터(`?v=YYYYMMDD`)를 붙여 접속한다.
* **배경:** Python `http.server`는 기본적으로 `Cache-Control` 헤더를 보내지 않아 실기기 브라우저가 이전 응답을 캐시로 사용한다. 이로 인해 코드 수정이 실기기에 반영되지 않은 것처럼 보이는 오진이 반복 발생했다.
* **규칙:** 서버 시작 시 `serve.py`(캐시 비활성화 커스텀 핸들러)를 사용한다. 상세 내용은 `serve.py` 참조.

## 픽셀 좌표 검증 원칙
* **Date:** 2026-07-11
* **Decision:** 정렬 관련 UI 작업 완료 후 "정렬 OK" 판단 시 `getBoundingClientRect()` 를 사용하여 핵심 요소들의 `left` / `right` 좌표를 실수치(px)로 첨부하는 것을 필수로 한다. "비슷함" / "대략 맞음" 판정은 허용하지 않는다.
* **배경:** 여러 라운드에서 육안 확인만으로 정렬을 통과 판정했다가 실기기에서 재차 오정렬이 발견되어 재작업이 반복됐다.
* **규칙:** 정렬 작업 시 검증 기준 요소(검색창, 탭 래퍼, 그룹 카드)의 `left` 목표 0px, `right` 목표 viewport width와 수치 일치 여부를 코드 또는 브라우저 콘솔로 확인 후 보고한다.

---

## 풀블리드 레이아웃 기본 방침 (모바일+태블릿)
* **Date:** 2026-07-11
* **Decision:** `calculator.html`의 모바일+태블릿(1024px 이하) 환경에서 검색창·카테고리 탭·그룹 카드는 **화면 좌측 0px에서 시작해 우측 끝(100vw)까지 완전히 채우는 풀블리드(Full-bleed) 레이아웃**을 기본으로 한다.
* **배경:** 헤더 로고 위치(약 20px 여백)와 콘텐츠 영역 여백이 달라 시각적 불일치가 발생했고, 감독 지시에 따라 콘텐츠는 풀블리드, 헤더 로고는 현행(약 20px) 유지로 최종 확정됐다.
* **규칙:**
  - `검색창`: 폭 100vw. 내부 텍스트 영역은 좌측 44px(돋보기 아이콘), 우측 16px padding 유지.
  - `카테고리 탭 래퍼`: 폭 100vw. 단, `.calc-tabs` 내부 좌우 padding 16px로 첫/마지막 탭 여백 확보.
  - `그룹 카드`: 좌우 margin 0, 좌우 border 제거. 내부 콘텐츠 padding-left/right 16px 유지.
  - `헤더`: 풀블리드 대상 아님. 기존 여백 유지.

## 재발 방지 원칙 7건 (2026-07-11 확정)

### [원칙 1] 커밋 전 git status 전체 파일 목록 체크
* **Date:** 2026-07-11
* **Decision:** 매 커밋 전 `git status`로 전체 변경 파일 목록을 먼저 표로 확인하고, 이번 작업 범위의 파일이 **전부 포함됐는지** 체크한 후 커밋한다.
* **배경:** `index.html` 변경사항이 여러 라운드 동안 커밋 사이클에서 누락되어 Netlify 실서버에만 옛 버전이 남아 있던 사고가 실제로 발생했다.
* **규칙:** `git add` 후 `git status`로 스테이징 목록을 재확인. 작업 지시에 언급된 파일이 전부 포함됐는지 체크리스트로 대조 후 커밋.

---

### [원칙 2] 커밋·push는 감독 명시적 승인 후에만 진행
* **Date:** 2026-07-11
* **Decision:** 커밋·push는 감독(사장님)의 명시적 **"확인 완료"** 또는 **"승인"** 문구가 있을 때만 진행한다. 코드 계산·자체 검증 결과만으로 임의 진행 금지.
* **배경:** 자체 검증 통과 후 감독 확인 없이 커밋을 진행했다가 미달 사항이 발견되어 재작업이 반복됐다.
* **규칙:** 작업 완료 보고 후 반드시 감독의 명시적 승인 대기. 승인 전 커밋 절대 금지.

---

### [원칙 3] 여러 파일 지시 시 파일별 개별 반영 확인
* **Date:** 2026-07-11
* **Decision:** 여러 파일에 걸친 지시는 각 파일별로 반영 여부를 **개별 확인**한다. 파일명이 명시돼도 다른 파일의 유사 로직·구조는 교차검증한다.
* **배경:** `calculator.html` 작업에 집중하는 동안 `index.html`·`catalog.html`의 유사 구조 변경이 반복적으로 누락됐다.
* **규칙:** 지시서에 명시된 파일 외에도 동일한 컴포넌트(헤더, 탭, 카드 등)를 공유하는 파일이 있다면 교차 확인 후 보고.

---

### [원칙 4] 로컬 서버 no-store + cloudflared 터널 우선 사용
* **Date:** 2026-07-11
* **Decision:** 로컬 개발 서버는 `Cache-Control: no-store` 헤더로 응답하고, LTE 등 Wi-Fi 외 네트워크에서도 확인 가능하도록 기본적으로 **cloudflared 터널을 우선 사용**한다.
* **배경:** Python `http.server` 기본 캐시 헤더 미설정으로 실기기 브라우저가 구버전을 캐시하여 코드 수정이 반영되지 않은 것처럼 보이는 오진이 반복됐다. Wi-Fi 의존 링크는 LTE 환경 감독 확인 불가.
* **규칙:** 서버 실행 시 `uv run python serve.py` 사용 (프로젝트 루트의 `serve.py` 참조). 실기기 링크는 cloudflared 터널 URL로 발급.

---

### [원칙 5] 정렬·여백 검증은 픽셀 좌표 수치 필수
* **Date:** 2026-07-11
* **Decision:** 정렬·여백 검증은 **"비슷함"이 아닌 픽셀 좌표 수치**(`getBoundingClientRect`)로 확인한다.
* **배경:** 육안 확인만으로 정렬 통과를 판정했다가 실기기에서 오정렬이 재발하여 여러 라운드 재작업이 발생했다.
* **규칙:** 정렬 작업 완료 후 `element.getBoundingClientRect().left` / `.right`를 콘솔에서 실측하여 목표 수치(left=0, right=viewport width 등)와 일치 여부를 수치로 첨부. "비슷함" 판정 금지.

---

### [원칙 6] 인라인 style 확인 우선
* **Date:** 2026-07-11
* **Decision:** 정렬·여백 이슈 발생 시 CSS `!important` 수정 전에 **HTML 인라인 `style=` 속성 존재 여부부터 확인**한다.
* **배경:** 인라인 `style=`은 CSS 명세상 `!important`보다 우선순위가 높다. `calculator-section`과 `global-search-container`의 인라인 스타일이 모든 미디어 쿼리 오버라이드를 무력화하여 정렬 오류가 반복됐다.
* **규칙:** 스타일 이슈 발생 시 `Select-String -Pattern 'style='` 또는 grep으로 해당 HTML 요소의 인라인 스타일을 먼저 점검.

---

### [원칙 7] 모바일+태블릿 풀블리드가 기본 방침
* **Date:** 2026-07-11
* **Decision:** 모바일/태블릿 UI 개선은 별도 명시 없는 한 두 브레이크포인트(768px 이하, 769~1024px) **모두 대상**이며, **풀블리드가 기본 방침**이다.
* **배경:** "모바일"이라고만 지시가 와도 태블릿도 동일하게 적용해야 하며, 콘텐츠 박스는 화면 끝까지, 내부 텍스트는 16px 안쪽 padding을 유지하는 것이 확정된 디자인 방향이다.
* **규칙:**
  - 박스(검색창·탭·카드): 화면 끝까지 (left=0, right=100vw)
  - 내부 텍스트·아이콘: 좌측 최소 16px(또는 아이콘 공간) padding 유지
  - PC(1025px+)는 영향 없음 — 별도 미디어 쿼리로 격리

# Architecture & UI Decisions

## UI Breakpoint & Optimization Policy
* **Date:** 2026-07-11
* **Decision:** 모든 UI 개선 작업(여백 조정, 헤더 축소, 모바일 전용 컴포넌트 등)은 별도 명시가 없는 한 **모바일(768px 이하)과 태블릿(769px ~ 1024px)** 두 브레이크포인트 모두에 동일하게 적용한다.
* **배경:** 모바일뿐만 아니라 태블릿(아이패드 등) 환경에서도 동일하게 일관된 고밀도 콤팩트 레이아웃을 제공하기 위함이다. PC 스타일은 **1025px 이상**의 환경에서만 독립적으로 작동한다.

## 인라인 style 우선순위 확인 원칙
* **Date:** 2026-07-11
* **Decision:** CSS 미디어 쿼리에서 `!important`를 사용해도 HTML 마크업의 인라인 `style=""` 속성보다 우선순위가 낮다. 스타일 적용이 예상대로 되지 않을 때는 **반드시 HTML 마크업의 인라인 스타일을 먼저 점검**한다.
* **배경:** 인라인 스타일은 CSS 명세상 선택자 우선순위 계산 밖에서 가장 높은 명시도를 가진다. 실제 작업에서 `.calculator-section`과 `.global-search-container`의 인라인 스타일이 `!important` 미디어 쿼리 전부를 무력화하여 좌우 여백이 남는 현상이 반복 발생했다.
* **규칙:** 스타일 변경 시 해당 HTML 요소에 인라인 `style=`이 존재하는지 grep으로 먼저 확인한 후 작업한다.

## 로컬 개발 서버 캐시 정책 (no-store)
* **Date:** 2026-07-11
* **Decision:** 로컬 개발용 Python HTTP 서버는 `Cache-Control: no-store` 헤더를 반드시 포함하여 서빙한다. 실기기(모바일)에서 확인 시 파일 수정 후 반드시 브라우저 강제 새로고침 또는 캐시 버스터(`?v=YYYYMMDD`)를 붙여 접속한다.
* **배경:** Python `http.server`는 기본적으로 `Cache-Control` 헤더를 보내지 않아 실기기 브라우저가 이전 응답을 캐시로 사용한다. 이로 인해 코드 수정이 실기기에 반영되지 않은 것처럼 보이는 오진이 반복 발생했다.
* **규칙:** 서버 시작 시 `serve.py`(캐시 비활성화 커스텀 핸들러)를 사용한다. 상세 내용은 `serve.py` 참조.

## 픽셀 좌표 검증 원칙
* **Date:** 2026-07-11
* **Decision:** 정렬 관련 UI 작업 완료 후 "정렬 OK" 판단 시 `getBoundingClientRect()` 를 사용하여 핵심 요소들의 `left` / `right` 좌표를 실수치(px)로 첨부하는 것을 필수로 한다. "비슷함" / "대략 맞음" 판정은 허용하지 않는다.
* **배경:** 여러 라운드에서 육안 확인만으로 정렬을 통과 판정했다가 실기기에서 재차 오정렬이 발견되어 재작업이 반복됐다.
* **규칙:** 정렬 작업 시 검증 기준 요소(검색창, 탭 래퍼, 그룹 카드)의 `left` 목표 0px, `right` 목표 viewport width와 수치 일치 여부를 코드 또는 브라우저 콘솔로 확인 후 보고한다.

## 풀블리드 레이아웃 기본 방침 (모바일+태블릿)
* **Date:** 2026-07-11
* **Decision:** `calculator.html`의 모바일+태블릿(1024px 이하) 환경에서 검색창·카테고리 탭·그룹 카드는 **화면 좌측 0px에서 시작해 우측 끝(100vw)까지 완전히 채우는 풀블리드(Full-bleed) 레이아웃**을 기본으로 한다.
* **배경:** 헤더 로고 위치(약 20px 여백)와 콘텐츠 영역 여백이 달라 시각적 불일치가 발생했고, 감독 지시에 따라 콘텐츠는 풀블리드, 헤더 로고는 현행(약 20px) 유지로 최종 확정됐다.
* **규칙:**
  - `검색창`: 폭 100vw. 내부 텍스트 영역은 좌측 44px(돋보기 아이콘), 우측 16px padding 유지.
  - `카테고리 탭 래퍼`: 폭 100vw. 단, `.calc-tabs` 내부 좌우 padding 16px로 첫/마지막 탭 여백 확보.
  - `그룹 카드`: 좌우 margin 0, 좌우 border 제거. 내부 콘텐츠 padding-left/right 16px 유지.
  - `헤더`: 풀블리드 대상 아님. 기존 여백 유지.
